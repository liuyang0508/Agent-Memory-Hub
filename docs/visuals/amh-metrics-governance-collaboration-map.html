Agent Memory Hub Capability Map

AMH 指标 / 治理 / 协作能力地图

这张图把 AMH 表达为本地优先的跨智能体可信上下文操作系统：协作入口产生证据和候选，治理层先维护共享事实源，指标层持续观测质量与风险，召回链路再按需把分层上下文注入给 Agent。

边界口径 raw conversation、MemoryItem、evidence、derived index、context view 和 adapter capability 分开画，避免把系统说成单一 RAG 或 transcript 仓库。
C协作层

不同 Agent 和工具不需要互相直连，统一通过本地共享大脑和受控接口复用上下文。

Agent adapters

Codex、Claude Code、Qoder、Wukong、Hermes 等按 adapter truth contract 暴露能力边界。

MCP / CLI / SDK / Web

读写、brief、search、doctor、review、Cockpit 和候选审批走统一工具面。

Lifecycle hooks

SessionStart、UserPromptSubmit、Stop、PreCompact、SubagentStart 等记录运行证据。

Resume & handoff

memory brief、开放 signal、最近 handoff 和关键 decision 支撑新会话承接。

Awareness injection

AGENTS.md、CLAUDE.md、rules、sidecar 让不同客户端理解同一套记忆纪律。

M指标层

指标不是装饰面板，而是治理动作的输入和协作可信度的证据。

Health / govern metrics

memory health、govern plan、category counts、maturity、retention 和 decay 风险。

Retrieval trace

search --explain、trace、context_firewall reason 解释为什么取、为什么不注入。

Data-flow read model

GET /api/data-flow 聚合 adapter events、verification、loop、gap、outcome、cohort。

Adapter verification

adapter doctor、install-verify、runtime evidence 和 verification record 分层判定。

Loop engineering ledger

runtime/loop-events.jsonl 记录目标、检查点、失败、验证和完成事件。

治理先行的共享大脑闭环

维护链路先决定什么可以进入、如何成熟、何时降权或复核；召回链路再按用户问题把可信上下文分层注入。

source-of-truth: Markdown MemoryItem indexes: rebuildable projection risky actions: review gate
01

维护完整链路

把协作过程中的证据、候选和质量信号转成可审计、可回滚、可复核的长期知识。

governance / maintenance
input 证据进入 conversation、resource、runtime event、feedback、adapter verification 进入证据层。
gate 写入漏斗 WriteService 统一执行 schema、audit gate、字段补全和质量 warning。
truth MemoryItem 落盘 items/mem-*.md 是长期事实源；raw transcript 不直接等于记忆。
index 派生索引 SQLite、FTS、vector、refs_graph、context view 是可重建投影。
plan 维护计划 govern plan / anti-drift 发现过期、摘要过长、漂移、矛盾、引用腐烂。
apply 安全执行 / 复核 summary rewrite 等低风险动作可 dry-run、snapshot、rollback；archive/delete/merge 进 review。

Shared Brain
可信事实层

AMH 的核心不是“多塞上下文”，而是把证据、长期知识、派生索引、治理状态和注入视图分层管理。

Raw conversation message 级原始证据，默认不自动注入 prompt。
Evidence / provenance refs、resource、extraction、commit、URL、memory link 支撑溯源。
Derived indexes FTS、vector、graph、runtime ledger 用来加速读取和观测。
Context views locator、overview、detail、context_pack 控制注入粒度。
02

召回时序链路

不是一句“召回”带过，而是从问题到过滤、融合、重排、扩展、防火墙和上下文注入的真实读取顺序。

retrieval / injection
Q 用户问题 query signal 先判断是否适合注入、有哪些强锚点。
filter 元数据过滤 project、type、tenant、tag、scope、sensitivity、status 先收窄候选。
parallel 全文 + 向量 BM25/FTS 与 vector 并行；向量退化时回到 BM25-only。
RRF 融合排序 RRF 融合多路候选，降低单一路径误召回风险。
score 重排与衰减 rerank、maturity、confidence、feedback、freshness、遗忘衰减参与评分。
expand 多样化 / 关联 MMR、Hopfield、graph expansion 负责去冗余和关联扩展。
firewall ContextFirewall 按来源、时效、敏感、预算、反馈、cohort 覆盖裁决 include/demote/exclude。
pack 分层上下文注入 context_pack 带 locator/overview/detail_uri/retrieve hint，不默认倾倒全文。
绿色实线：治理和维护写回主路径。
青色实线：召回、过滤和上下文注入路径。
琥珀实线：指标观测反哺维护计划。
虚线：需要复核、门禁或条件启用的路径。
