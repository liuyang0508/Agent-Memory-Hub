把每次智能协作，写成下一次的起点

Agent Memory Hub 不是 transcript 仓库，也不是单个 Agent 的插件。它是一个本地优先、可追溯、可治理的共享第二大脑：证据进入证据层，长期结论进入 Markdown，索引只是可重建投影，注入前必须经过上下文防火墙。

革命点不在“记住”，而在“可信接力”

把 AI 协作从一次性 prompt，变成有证据、有边界、有治理、有下一步的工程系统。

从聊天记录到证据合同

原始对话落在 evidence；MemoryItem 只保存可复用结论。证据、记忆、索引分层，避免上下文污染。

从黑箱 RAG 到可解释检索

BM25、向量、RRF、MMR、衰减、反馈、状态、运行证据共同排序；retrieval trace 能说明为什么被召回。

从单 Agent 到共享第二大脑

Claude Code、Codex、Hermes、Qoder、Wukong 和 Open* 适配器共享同一个本地脑池，但按真实能力边界接入。

核心链路

每一步都有产物、有回退、有诊断，不把“智能”藏进不可审计的魔法里。

1

采集证据

conversation、resources、extractions 保留来源。

2

写入漏斗

WriteService 负责 schema、审计、质量和索引。

3

投影索引

Markdown 是事实源；SQLite、FTS、向量、图谱可重建。

4

召回融合

多路召回合并排序，再叠加时效、反馈和风险。

5

上下文防火墙

过期、低置信、跨范围、危险上下文先被挡住。

6

Agent 接力

hooks、MCP、provider、文件上下文按工具能力注入。

全代码能力审视

按源码模块和真实公开接口梳理，不按愿景写功能。

能力层已经具备为什么关键
接口面28 个 MCP tools、CLI、SDK、91 条 Web/API/WS routes（由 surface lock 稳定锁定）、三日数据流转、记忆链路追踪、实时事件、Webhooks、备份、图谱、Cockpit、compression gate 和 ML/DL advisory gate。同一个脑池能被脚本、Agent、网页和工具链同时使用。
写入面CLI/MCP/Web create、merge、Hermes remember、proactive approval 均走 WriteService；失败索引写入进入 pending 可修复队列。让新增长期记忆经过统一 schema、审计、质量和索引流程。
检索面BM25、向量、RRF、rerank、MMR、graph expansion、Hopfield、temporal、decay、supersession、status、trace、budget。不是“相似度最高就注入 prompt”，而是可治理的召回系统。
上下文面context loading、context packing、query signal、context firewall、injection cohorts、injection feedback。让自动注入可解释、可回退、可按任务范围收敛。
治理面audit、outbound audit、drift、duplicates、contradictions、citation rot、staleness、tiering、maturity、review queue、evolve。第二大脑不是越写越乱，而是能老化、合并、复核、晋级。
证据面conversation store、resource store、import service、harvester、transcript reader、Obsidian integration、resource sidecar。长期记忆能追到原始证据，资源和抽取不直接污染 MemoryItem。
适配面16 个 adapter：11 verified、4 install-ready、1 wip；Codex 和 QoderWork 当前 verified，Claude Code、OpenClaw、Qoder、Wukong 仍为 install-ready。不把所有 Agent 画成同一种能力，谁能 hooks、谁能 MCP、谁是 provider、谁只有兼容文件旁路，必须分开。
验证面public surface lock、conformance、unit、perf、dashboard parse、route lock；成功数字必须来自当前验证输出，不能写成静态宣传数字。接口不会悄悄漂移，README 数字和 Web 可视化不再靠手填。
