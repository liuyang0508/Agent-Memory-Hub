我们的四个差异轴

不是每个单点都独占。可 defend 的是组合：可审计文本源、写前治理、跨 agent 协议、多模态派生层不污染原始事实。

Markdown 是事实源

memory item 以 md + frontmatter 落盘。SQLite、向量、图关系是派生索引，删掉可重建。这是对 cloud store、binary engine、压缩层的反向选择。

写前治理，不是事后审计

skill audit / safety gate 在写入前阻止高风险内容进入脑池。rohitg00 的 audit trail 很强，但更偏记录发生过什么。

MCP-native 多智能体协作

稳定核心工具 + 分层扩展工具，面向 Claude Code、Codex、Cursor、Wukong 等多个 agent，而不是某个框架或单一产品内存。

Resource / Extraction / Memory 三层

多模态未来不直接写 memory。原件保留，抽取可重建，结论可审计。借 OpenViking/Mem0/Supermemory 的抽取思路，但保留本地可追溯性。

Obsidian 结合层

Obsidian 不是另一个 memory 后端，而是人类可读、可编辑、可审计的 vault 界面。源事实仍在 brain pool 的 Markdown item；wiki / entity / health 页面是派生视图，生成错了可以删掉重建。

Item 级双向同步

每条 MemoryItem 导出成 Obsidian 兼容 Markdown：YAML frontmatter、正文、tags、refs、相关记忆 wikilinks。人工在 vault 里修正文档后，可再 import 回 brain pool 并重建索引。

agent_runtime_kit/itemsmd + frontmatter 源数据
export生成 vault note
human edit人工修标题、来源、链接
import回写合法 item
reindexFTS / vector 可重建

LLM-Wiki 派生视图

obsidian-export --wiki 会额外生成 index.md、log.md、health/report.md 和 entity 页面。它们方便人在 Obsidian 里浏览全局结构，但不会进入 items 池污染检索。

index.md类型、项目、热度统计
log.md最近写入维护流
health孤岛、缺来源、过期
entities人/项目/主题索引
derived删掉可再生成

人类审计，不绕过 gate

Obsidian 负责让人看得懂、改得动、连得上；before-write 仍负责防止高风险内容进 brain pool。未来多模态资源也应先产生 extraction evidence，再在 Obsidian 中暴露给人复核。

review人工确认事实和来源

audit写前安全和质量门
memory只保存确认结论
agents多 CLI 共享召回

融合路线：取其精华，保留我们的硬边界

最终目标不是“也支持图片/PDF/音视频”，而是让任何资源都通过可审计派生层进入 memory。原件不丢，抽取可重跑，结论可追责。

规则 1：原始资源不是 memory 结论

图片/PDF/音视频可以进入 Resource Registry；memory 只能保存“经过抽取和审核的事实/决策/信号”。

规则 2：抽取结果必须可追溯

每条派生文本记录 extractor、版本、时间、置信度、原件 hash/source_ref。未来模型升级时可以重跑。

规则 3：检索仍以文本事实为主

短期不引入复杂多模态向量库；先把 OCR/VLM/ASR 结果变成高质量 body，再走现有 BM25 + vector + graph。

v0 Resource Contract

定义 Resource、Extraction、MemoryItem 引用关系。支持落盘、hash、source_ref、extractor metadata，不做重型模型依赖。

  • 借 OpenViking 路径化资源
  • 保留 md source-of-truth

v1 Image + PDF Gate

先做最容易验证的图片和 PDF：无“视觉事实/文本抽取”摘要则 warning 或拒写，避免 Image # 噪声污染检索。

  • 借 Mem0 vision extraction
  • 借 Supermemory OCR 思路

v2 Audio / Video Parser

把 ASR transcript、关键帧描述、章节摘要接入同一 Extraction Contract，而不是另开一套记忆 schema。

  • 资源级 metadata
  • 时间戳引用

v3 Review + Graph

把资源、抽取、memory item 和 refs_graph 接起来，支持人工确认、confidence 调整、跨资源实体关系。

  • 借 Zep temporal graph
  • 比 cloud store 更可审计
